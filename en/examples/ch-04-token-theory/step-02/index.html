Resolution vs reference

The same set of tokens compiled two ways. In resolved mode, every variable carries its literal value. In referenced mode, the chain is preserved with var(). Both work, but they behave very differently when you try to change the primitive at runtime.

var(--button-background-primary) reaches the primitive through the chain

Move the colour picker. In referenced mode the swatch follows, because the component token ultimately resolves to var(--color-blue-500) at paint time. In resolved mode the swatch stays put, because the component value is already baked into a literal hex at build time.
